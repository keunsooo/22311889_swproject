## 1. Introduction
 1) Summary
사회에서는 학업, 업무, 개인 일정 등 다양한 활동이 동시에 이루어지면서 효율적인 시간 관리의 중요성이 지속적으로 증가하고 있다.
이에 따라 캘린더(Calendar) 및 투두리스트(To-Do List) 애플리케이션의 활용도 역시 높아지고 있다.
하지만 기존 서비스들은 일정 관리와 할 일 관리 기능이 분리되어 있는 경우가 많으며, 사용자 맞춤형 UI 및 통합 관리 기능이 부족하다는 한계가 존재한다.
사용자는 캘린더 기능을 통해 월별, 주별, 일별 일정을 한눈에 확인할 수 있으며, 일정 등록, 수정, 삭제 기능을 사용하여 자신의 스케줄을 효율적으로 관리할 수 있다.
투두리스트 기능을 통해 사용자는 해야 할 작업을 추가하고 관리할 수 있다. 각 작업은 완료 여부를 체크할 수 있으며, 우선순위를 설정하여 중요한 작업을 먼저 처리할 수 있도록 구성한다.
사용자 편의성을 높이기 위해 직관적이고 간단한 인터페이스(UI)를 적용하였으며, 테마 색상 변경 기능을 통해 사용자 맞춤형 환경을 제공한다.
따라서 사용자는 자신의 취향에 맞는 화면 구성을 설정할 수 있으며, 보다 편리하고 쾌적한 환경에서 애플리케이션을 사용할 수 있다.

 3) The features of project 
To Do Calendar는 일정(Calendar)과 할 일(To-Do List)을 하나의 시스템에서 통합적으로 관리할 수 있는 애플리케이션이다.
사용자는 직관적인 인터페이스를 통해 일정을 등록하고 할 일을 관리할 수 있으며, UI 커스터마이징 기능을 통해 개인화된 환경을 구성할 수 있다.

## 3. IDomain analysis

| 이름 | 설명 |
|---|---|
| User | 사용자 클래스이다. 회원가입 및 로그인 기능을 수행하며 일정, To-Do, UI 설정 정보를 가진다. |
| Calendar | 사용자의 일정 정보를 관리하는 클래스이다. 여러 개의 일정(Schedule)을 포함한다. |
| Schedule | 일정 클래스이다. 일정 제목, 날짜, 시간, 메모 등의 정보를 가진다. 사용자는 일정을 추가, 수정, 삭제할 수 있다. |
| ToDo | 할 일 클래스이다. 해야 할 작업의 제목, 상태, 우선순위 등의 정보를 가진다. 완료 여부를 체크할 수 있다. |
| Theme | UI 테마 클래스이다. 사용자가 선택한 색상 테마 정보를 저장한다. |
| BackgroundImage | 배경 이미지 클래스이다. 사용자가 선택한 배경 이미지 정보를 저장한다. |
| Notification | 알림 클래스이다. 일정 시간이나 할 일 마감 시간에 사용자에게 알림을 제공한다. |
| Authentication | 사용자 인증 클래스이다. 로그인, 로그아웃 및 사용자 인증 처리를 담당한다. |
| Settings | 사용자 설정 클래스이다. UI 설정, 알림 설정 등 애플리케이션 환경설정을 저장한다. |
| ScheduleManager | 일정 관리 클래스이다. 일정 생성, 수정, 삭제 및 조회 기능을 담당한다. |
| ToDoManager | To-Do 관리 클래스이다. 할 일 추가, 완료 처리, 삭제 기능을 담당한다. |
| CustomizationManager | UI 커스터마이징 관리 클래스이다. 테마 색상 및 배경 이미지를 변경하고 저장하는 기능을 담당한다. |

# 5. Glossary

| Term | Description |
|---|---|
| To Do Calendar | 일정 관리와 To-Do List 기능을 통합한 애플리케이션 |
| Schedule | 사용자가 특정 날짜와 시간에 등록하는 일정 정보 |
| To-Do | 사용자가 수행해야 하는 작업 또는 할 일 |
| Theme | 사용자가 설정할 수 있는 애플리케이션의 색상 테마 |
| Background Image | 사용자가 선택하여 적용할 수 있는 배경 이미지 |
| Calendar View | 월별 또는 날짜별 일정 정보를 보여주는 화면 |
